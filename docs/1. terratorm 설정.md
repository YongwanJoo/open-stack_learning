# 1. Terraform 초기 설정 및 OCI 인프라 배포

이 문서는 Oracle Cloud Infrastructure (OCI) 위에 OpenStack 구동을 위한 기본 인프라를 Terraform으로 구성하고, 배포된 가상 머신에 SSH로 접속하기까지의 과정을 정리합니다.

---

## 1. OCI 인증 환경 변수 설정
`main.tf`에 민감한 인증 정보를 하드코딩하지 않고 환경 변수로 관리하기 위해, 로컬의 `.oci/config` 파일을 읽어 `TF_VAR_*` 형태로 설정해 주는 쉘 스크립트(`set_env.sh`)를 사용

```bash
# 환경 변수 스크립트 실행 권한 부여 및 적용
cd terraform
chmod +x set_env.sh
source set_env.sh
```

## 2. Terraform 초기화 및 의존성 설치
OCI Provider 플러그인을 다운로드하고 작업 공간을 초기화하기 위해 `terraform init` 명령어 실행
이 과정에서 생성되는 `.terraform.lock.hcl` 파일은 플러그인 버전을 고정해 주므로 깃허브에 함께 커밋하여 버전 관리

```bash
terraform init
```

## 3. SSH 공개키 생성
생성될 OCI 가상 머신(VM)에 원격 접속하기 위해서는 SSH 키 쌍이 필요
로컬 PC에 키가 없다면 아래 명령어를 통해 SSH 키를 생성 (생성된 공개키 `id_rsa.pub` 파일이 Terraform 코드를 통해 VM에 주입)

```bash
# RSA 방식의 4096비트 SSH 키 생성
ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa -N ""
```

## 4. 인프라 배포 (Terraform Apply)
모든 준비가 완료되면 코드를 실행하여 실제 OCI 환경에 자원(VCN, Subnet, Internet Gateway, Route Table, Compute Instance)을 배포

```bash
terraform apply
```
> 배포가 완료되면 터미널 결과창(Outputs)에 가상 머신의 공인 IP(`instance_public_ip`)가 출력됨

## 5. 생성된 인스턴스에 SSH 접속
출력된 공인 IP와 앞서 생성한 개인키(`id_rsa`)를 사용하여 배포된 Ubuntu 인스턴스에 접속

```bash
# 접속 명령어 형식
ssh -i ~/.ssh/id_rsa ubuntu@<인스턴스_공인_IP>

# 실제 접속 예시
ssh -i ~/.ssh/id_rsa ubuntu@your_IP
```
접속 시 지문을 신뢰하겠냐는 프롬프트(`Are you sure you want to continue connecting?`)가 나오면 `yes`를 입력하여 접속을 완료합니다. 성공적으로 접속되면 `ubuntu@openstack-node-01:~$` 프롬프트를 확인할 수 있습니다.
